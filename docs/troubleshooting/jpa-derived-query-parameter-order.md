## Spring Data JPA 파생 쿼리 조건 순서와 인자 순서 불일치 문제

### 문제
조건이 많아질수록 파생 쿼리 메서드명이 길어졌고, 조건 순서와 파라미터 전달 순서에 대한 의존성이 커졌습니다.  
그 과정에서 `novelId`와 `episodeId`가 잘못 전달되어 의도와 다른 조회 결과가 발생했습니다.  
두 값이 모두 `Long` 타입이어서 컴파일 단계에서 즉시 확인되지 않았습니다.


### 해결
메서드명에 선언된 조건 순서와 파라미터 순서를 맞추도록 수정해 의도한 조건으로 조회가 수행되도록 변경했습니다.

### 결과
복잡한 조건 조회를 파생 쿼리만으로 구현하면 유지보수가 어려워질 수 있음을 경험했으며, Spring Data JPA 파생 쿼리는 메서드명과  
인자 순서가 함께 의미를 가지므로 두 요소를 일관되게 관리해야 한다는 점을 알게 되었습니다.
