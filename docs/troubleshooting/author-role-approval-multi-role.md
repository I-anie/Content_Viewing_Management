## 작가 권한 승인 시 기존 사용자 권한이 사라지던 문제 해결

### 문제
초기에는 사용자 권한을 단일 값으로 관리해, 작가 권한 요청 승인 시 기존 `USER` 권한이 `AUTHOR`로 덮어씌워지는 문제가 있었습니다.  
그 결과 작가가 된 사용자는 일반 사용자 권한이 필요한 기능에 접근하지 못했고, 본인 작품 외 유료 회차 열람에도 제약이 발생했습니다.

### 해결
권한을 여러 개 보관할 수 있도록 구조를 변경하고, 중복 저장을 방지하기 위해 `Set`으로 관리했습니다.  
이를 통해 기존 `USER` 권한을 유지한 채 `AUTHOR` 권한을 추가할 수 있도록 개선했습니다.

### 결과
작가 권한 승인 이후에도 일반 사용자 기능을 계속 사용할 수 있게 되었고, 권한과 비즈니스 로직이 의도한 방식대로  
동작하도록 개선했습니다.
