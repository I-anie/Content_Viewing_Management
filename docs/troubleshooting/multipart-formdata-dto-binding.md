## `multipart/form-data` 요청 처리 시 DTO 바인딩 문제

### 문제
`multipart/form-data` 요청을 처리하는 과정에서 DTO 필드가 정상적으로 바인딩되지 않는 문제가 있었습니다.  
요청은 정상적으로 전달되었지만, 컨트롤러에서 DTO 값이 비어 있는 상태로 들어와 이후 로직이 올바르게 동작하지 않았습니다.

### 해결
원인을 확인한 결과, `@ModelAttribute`는 setter 기반으로 요청 값을 주입하는데 DTO에 setter가 없어 바인딩이 실패하고 있었습니다.  
DTO에 `@Setter`를 추가해 요청 값이 정상적으로 주입되도록 수정했습니다.

### 결과
`multipart/form-data` 요청에서도 DTO 필드가 정상적으로 바인딩되어 업로드 및 입력 처리 로직이 의도한 대로 동작하도록 개선했습니다.  
이 경험을 통해 `@ModelAttribute` 바인딩 방식과 DTO 구조를 함께 점검해야 한다는 점을 알게 되었습니다.
