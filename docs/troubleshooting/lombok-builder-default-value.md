## Lombok Builder 사용 시 필드 기본값이 반영되지 않던 문제

### 문제
필드에 기본값을 지정했음에도 `@Builder`로 객체를 생성할 때 초기값이 반영되지 않는 문제가 있었습니다.  
그 결과 기본값이 설정되어야 할 필드가 `null` 또는 의도하지 않은 값으로 생성되었습니다.

### 해결
원인을 확인한 결과, Lombok Builder는 필드 초기화 값을 자동으로 보장하지 않았습니다.  
기본값이 필요한 필드에 `@Builder.Default`를 적용해 빌더 생성 시에도 초기값이 유지되도록 수정했습니다.

### 결과
Builder를 사용하더라도 기본값이 필요한 필드가 의도한 값으로 초기화되도록 개선했습니다.  
이 경험을 통해 Builder 패턴에서는 필드 기본값 처리 방식까지 함께 고려해야 한다는 점을 알게 되었습니다.
